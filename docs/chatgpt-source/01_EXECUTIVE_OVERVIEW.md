# Executive Overview

## Vision
Create a modern, modular taxi dispatch SaaS that replaces outdated operator software and becomes the operating system for independent taxi companies and regional taxi networks.

The system should combine:
- booking capture,
- dispatch and fleet control,
- driver operations,
- partner-company collaboration,
- white-label web lead generation,
- AI-assisted automation.

## Why this matters
Many taxi firms still rely on fragmented tools, phone-heavy workflows, basic dispatch software, spreadsheets, and manual coordination. Smaller firms especially need a system that is:
- easier to use,
- cheaper to adopt,
- more flexible,
- more web-native,
- better at automation,
- suitable for both local work and airport transfers.

## Core commercial idea
Start with one real operator use case, then expand to a multi-tenant SaaS:
- each taxi company gets its own branded operational environment,
- websites can feed bookings directly into dispatch,
- partner companies can share jobs,
- substitute drivers can be managed cleanly,
- pricing scales by operational complexity.

## Strategic differentiators
1. Built for real operator workflows, not enterprise-only fleets.
2. Supports substitute / external drivers and partner fulfilment from the outset.
3. Integrates with SEO-driven booking websites.
4. AI can be layered in gradually rather than forcing a complex v1.
5. Modular enough for parallel AI-agent development.

## Product principle
Keep the business flow simple:
Lead or booking comes in -> quote / fare rules applied -> booking confirmed -> dispatch decision made -> driver executes -> operator monitors -> finance and reporting follow.
